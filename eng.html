Sound photo: capture a much more playful photo with sound embedded in it. The sound is store in JPEG file’s exif APP2 section not a separating file, so it is much easier to transfer and share among difference devices.

Realtime face beauty: apply various effects like Distortion, Vignette, Depth of field, etc. on camera’s preview/capture frame buffer (YUV format) before sending it to graphics module to complete rendering and compositing. There are mainly 2 different solutions to achieve this, both of them are implemented, compared and finally solution B was chosen since it met our requirements from both performance and power aspects:

  • A: GPU/openGL This solution is much power consuming and device becomes hot quickly, because it uses OpenGL ES2.0 shader to do the YUV-RGB conversion first and then uses GL to render the RGB frame on GLSurfaceView.
  • B: CPU Just need to get the backend graphics buffer of the SurfaceView and copy the YUV frame buffer to the backed graphics buffer directly without color conversion. Since this solution use CPU to do compositing, it consumes less power than solution A, and thermal won’t increase quickly.

Camera performance and efficiency improvement:

It’s not an easy job, cause any changes should not involve degrades, but it worth, finally our camera app launches much quickly than before even for cold boot case.

#01: Java Bytecode Class File Encryption Based on Java Native Interface

Most Obfuscators only rename the identifiers in java class file. As such, it does nothing but makes the files decompiled by Java Decompilers difficult to read. Considering these limits, I overwrite the ClassLoader and use Java Native Interface to convert byte array into java class, because the main part is written in C, this approach puts the security on a level comparable to software written in a native language.
